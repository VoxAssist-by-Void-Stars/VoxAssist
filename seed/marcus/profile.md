# Marcus Bell — Profile

## About
Embedded and systems developer, 6 years. I work close to the metal — firmware, real-time systems,
drivers. I value **deterministic, well-documented behavior** over speed of delivery. If it isn't
tested and documented, it isn't done.

## How I like to work
- **Docs-and-tests-first.** I write the README and the test plan before the implementation.
- Cautious about changing system state — I confirm the evidence supports an action before running it.
- Precise specs and clear ownership boundaries. I dislike surprise refactors in code I own.
- Methodical and steady — I'd rather be right than first.

## Conventions
- Signed git commits, small and atomic.
- Thorough code review — I will read your diff line by line.
- Explicit error handling at boundaries; no silent failures.
- Integrate over MQTT or HTTPS with documented contracts.

## Hobbies
Road cycling (long weekend rides), building mechanical keyboards (I solder my own), and amateur radio.
